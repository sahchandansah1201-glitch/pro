# DESIGN.md

Register: design / UI context for Codex.

## UI tone and component style

Продуктовый клинический UI, не маркетинговый. Спокойный, плотный, операциональный.
Рабочее место врача — desktop-first, оптимизировано под 1440px+. Пациентские и
бот-потоки — mobile-first и максимально простые.

Базовые библиотеки: React + TypeScript, Tailwind CSS, shadcn/ui (через
`src/components/ui/*`), Lucide icons. Анимации сдержанные; при необходимости —
framer-motion, точечно.

## Layout conventions

- App shell: фиксированный сайдбар (`AppSidebar`) + верхний тулбар + рабочая
  область. Тёмный сайдбар (`--sidebar-*`), светлый контент.
- Страницы: `PageHeader` (h-page + подзаголовок), затем секции на карточках
  `surface-card` без двойных рамок.
- Тулбары фильтров/действий — `surface-toolbar` (приглушённый фон, тонкая
  граница), визуально отделены от контентных карточек.
- Секции внутри карточки начинаются `section-bar` — заголовок с акцентной левой
  полосой и тонкой нижней границей.
- Плотные строки: `row-grid` (ФИО | мета | статус | действие), без потоков «·».
- Таблицы: `data-table`, sticky header, hover на строке, плотные ячейки.
- Split-вью для рабочего места визита: фиксированные панели, вкладки внутри.
- Никаких hero-секций, центральных «landing»-блоков, нестед-карточек.

## Typography and density

Шрифты: **IBM Plex Sans** (UI и заголовки), **IBM Plex Mono** (код, идентификаторы,
табличные числа через `tabular-nums`). Не использовать Inter/Poppins/serif.

Шкала (см. `src/index.css`):

- `h-page` 20/600, `h-page-sub` 13 muted.
- `h-section` 15/600, `h-section-hint` 12 muted.
- `text-row` 14/relaxed — строки списков и таблиц.
- `text-meta` 13 muted — вторичная информация.
- `kpi-value` 28/600 tabular — крупные числа дашборда.
- Заголовки таблиц — 12, medium, **без uppercase**.

База контента — 14px, `line-height: 1.5`. Поддерживается пользовательский zoom
браузера (rem-base 16px на html). Длинные кириллические подписи не должны
обрезаться — использовать перенос, а не `truncate`, в клинических колонках.

## Color and state usage

Все цвета — HSL, через семантические токены `index.css` + `tailwind.config.ts`.
**Никаких прямых `text-white`, `bg-black`, hex в компонентах.**

Палитра:

- Поверхности: `--background` / `--surface` / `--surface-muted` / `--surface-sunken`.
- Акцент: один сдержанный clinical teal (`--primary`, `--primary-soft`).
- Семантика: `--info`, `--success`, `--warning`, `--destructive`.
- **Risk tokens** (`--risk-low|moderate|high|urgent`) — используются **только**
  вместе с текстовой меткой через `<RiskBadge />`. Цветом без подписи риск
  никогда не передаётся (доступность и медицинская безопасность).

Состояния:

- Hover/focus — через `--ring` (teal), видимый focus-ring обязателен.
- Disabled — через `text-muted-foreground` + `opacity`, не серым hex.
- Тени минимальны: `--shadow-sm` для карточек, `--shadow-overlay` для попапов.
- Радиусы — дисциплинированные (`--radius` 0.375rem), не «игрушечные».

## Accessibility expectations

- Контраст AA минимум для текста и иконок-смыслоносителей.
- Видимый focus-ring на всех интерактивных элементах.
- Tap target ≥ 44px на мобильных (есть e2e: `*-tap-target.pw.ts`).
- Никаких горизонтальных скроллов на core-страницах (e2e: `*-no-hscroll.pw.ts`).
- Семантика: правильные `<button>` / `<a>`, `aria-label` для иконок,
  `aria-live` для динамических статусов оператора/релиза.
- Risk и статусы — всегда дублируются текстом.
- Поддержка клавиатурной навигации в таблицах, тулбарах, диалогах.
- Скрин-ридер: осмысленные подписи у полей формы, ассоциированные `<label>`.

## Dashboard patterns

- KPI-плитки: `kpi-value` число + `kpi-label` подпись, без декоративных иконок-фонов.
- Графики — сдержанные, монохром + один акцент; легенда текстом.
- Фильтры в `surface-toolbar` сверху, состояние пустого списка — через
  `ListEmptyState` со спокойной формулировкой.
- Пагинация — `ListPagination`, числовая, без «магических» бесконечных скроллов
  в админских таблицах.

## Table patterns

- `data-table` с sticky-header.
- Колонки: ключевая сущность слева (ФИО/идентификатор), мета по центру, статус
  и действия справа (`row-action` justify-self-end).
- Числовые колонки — `tabular-nums`, выравнивание по правому краю.
- Сортировка — индикаторы стрелок, `aria-sort`.
- Пустое состояние — `ListEmptyState` с подсказкой действия, без иллюстраций.
- Длинный текст — перенос; обрезка только для технических id.

## Form patterns

- Вертикальные формы, метка над полем, helper-text 12 muted под полем.
- Ошибки — `--destructive`, текстом + иконкой; не только цветом.
- Группы полей — `fieldset` с тонкой границей или секцией внутри карточки.
- Кнопки: первичная — `primary`, вторичная — `secondary` / `ghost`.
  Деструктивные действия — `destructive` с подтверждением в диалоге.
- Опасные действия (отправка отчёта, удаление) — disabled до выполнения
  предусловий + явный helper-text почему disabled.

## Dialog patterns

- shadcn `Dialog` / `AlertDialog`. Модалки — короткие, с одной задачей.
- Заголовок — `h-section`, описание — `text-meta`.
- Кнопки внизу справа: вторичная «Отмена» слева от первичной.
- `AlertDialog` — для деструктивных и медицински значимых подтверждений
  (например, отправка отчёта пациенту).
- Drawer/Sheet — для побочных потоков (детали записи, фильтры на мобильном).

## Russian clinical UI copy rules

- Спокойный, конкретный, медицински безопасный регистр. Без маркетинга,
  без восклицаний, без эмодзи в продуктовом UI.
- Никогда не утверждать диагноз в пациент-фейсинг тексте: использовать
  «требуется осмотр врача», «рекомендована консультация», «нужно проверить очно».
- ИИ-поддержка подписывается явно: «поддержка по риску», «вспомогательный
  сигнал», не «диагноз».
- Кнопки — глагол + объект: «Сохранить заключение», «Отправить отчёт пациенту»,
  «Создать визит». Без «ОК» в значимых действиях.
- Статусы — короткие существительные: «Черновик», «Готово», «На проверке».
- Даты — локаль `ru-RU`, формат «дд.мм.гггг», время — 24-часовое.
- Числа — пробел как разделитель тысяч, запятая — десятичный разделитель.
- Термины: «дерматоскопия», «образование», «визит», «заключение», «отчёт»,
  «маршрутизация», «согласие». Не «кейс», «ассессмент», «реквест».

## Visual patterns to avoid

- Hero-секции, центрированные «landing»-блоки, крупные маркетинговые карточки.
- Декоративные градиенты, пурпур-на-белом, неоновые акценты.
- Стеклоэффекты, тяжёлые тени, скевоморфизм.
- Двойные рамки (карточка в карточке с двумя бордерами).
- Иконки без подписи там, где они несут смысл (особенно в риске и статусах).
- Цвет как единственный носитель смысла (нарушение a11y и медицинской
  безопасности).
- Эмодзи и игривые иллюстрации в клиническом UI.
- Шрифты Inter/Poppins/Roboto как основной — мы используем IBM Plex.
- Бесконечный скролл в административных таблицах.
- Длинные tooltip-объяснения вместо ясной подписи поля.
- Анимации, отвлекающие от данных; параллакс, авто-карусели.
