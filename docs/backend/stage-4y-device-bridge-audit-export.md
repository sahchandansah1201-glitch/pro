# Stage 4Y — Device Bridge audit export

Stage 4Y adds a safe CSV export flow for Device Bridge command audit evidence.
The export is generated by the self-hosted backend from PostgreSQL audit
projection data and is downloaded from the system UI without exposing command
payloads or worker internals.

## Scope

- Backend endpoint: `GET /api/v1/device-bridge-worker/audit/export`.
- Frontend surface: `/sys/devices`, button `Экспорт audit CSV`.
- OpenAPI: `/openapi.stage4y.json`.
- CI/preflight: `npm run preflight:stage4y`.

## Backend contract

The export endpoint requires `system_admin` through the existing ops scope. It
uses the safe Stage 4X command audit projection, then renders CSV content with
only operational columns:

- event id;
- created timestamp;
- action;
- command id;
- bridge code;
- device serial;
- command type;
- status;
- attempt count;
- lifecycle revision;
- replay policy;
- source replay command id;
- correlation id.

The CSV intentionally excludes `metadata_json`, `payload_json`, `result_json`,
worker tokens, raw storage paths, patient names, signed URLs, and browser
hardware API details.

Audit event:

- `device_bridge.command.audit.export`.

## Frontend behavior

When a self-hosted session is present, `/sys/devices` shows the Stage 4Y export
button in the `Device Bridge command audit and replay` section. Clicking it
calls the backend export endpoint, downloads the CSV using the backend-provided
filename, and announces the result in the existing page status message.

The browser never constructs audit rows from raw command payloads. It receives
only the backend-owned safe export payload.

## Self-hosted product boundary

This is the self-hosted product boundary for Stage 4Y.

The deployable product remains one owned stack:

- frontend served by the bundled app;
- backend Node service;
- PostgreSQL database;
- local object storage;
- local Device Bridge worker process.

No Supabase runtime, Edge Functions, managed database calls, browser WebUSB,
WebBluetooth, WebSerial, or third-party audit/export service is added.
