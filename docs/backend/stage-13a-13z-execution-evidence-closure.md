# Stage 13A-13Z - Execution evidence closure

Stage 13A-13Z closes the Stage 12A-12Z execution evidence bundle. It is a
repository-only process batch: no production runtime, database, device bridge,
clinical workflow, or browser behavior changes.

## Purpose

The batch turns the Stage 12 evidence package into a closure contract. The
closure prevents these recurring failures:

- saying a batch is ready before the Pull request is merged;
- sending a Lovable prompt before local `main` contains the merge;
- treating a missing Lovable file as a code conflict before checking sync state;
- relying on chat memory instead of repository files and commands.

## Previous Evidence Regression

Stage 13A-13Z must run `npm run preflight:stage12a-12z`. This makes
the previous evidence bundle a hard regression gate instead of a historical
note.

## GitHub Closure

The valid lifecycle is:

1. Create branch.
2. Commit the batch.
3. Push branch.
4. Open Pull request.
5. Wait for checks.
6. Mark ready only after checks pass.
7. Squash merge.
8. Verify local `main`.
9. Send Lovable prompt.

## Lovable Closure

The prompt is generated by
`scripts/stage13a-13z-execution-evidence-closure.mjs`. It must be sent only
after merge into `main`. If Lovable reports missing files, diagnose sync state
before changing code.
