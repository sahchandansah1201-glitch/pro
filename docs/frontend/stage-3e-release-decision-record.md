# Stage 3E — Release decision record for auth/assets readiness

## 1. Purpose

- Final go/no-go decision record for promoting the auth/assets readiness slice.
- Documentation only. Does not change runtime behavior.
- Complements Stage 3A deploy, Stage 3B rollback drill, Stage 3C smoke, Stage 3D incident response.

## 2. Release decision

- Record one of: GO, NO-GO, DEFERRED.
- Record target ref / build artifact identifier (commit SHA, no secrets).
- Record decision timestamp.
- Record the deciding owner (placeholder; real name kept outside repo).

## 3. Required evidence

- `npm run preflight:auth-assets` ends with `[preflight-auth-assets] OK`.
- `node scripts/check-no-deno-locks.mjs` passes.
- `frontend-auth-assets` workflow is green on the target ref.
- `auth-assets-smoke-skip` workflow is green or skipped as expected.
- Stage 3C production smoke completed or explicitly deferred (docs/frontend/stage-3c-production-smoke.md).
- Stage 3D incident response doc is linked and available (docs/frontend/stage-3d-incident-response.md).
- `package-lock.json` is preserved (not deleted, not regenerated by hand).
- No `deno.lock` files exist anywhere in the repository.

## 4. Go criteria

- Auth routing works (`/login` real auth signs in a doctor user).
- Protected-route redirect works for anonymous users.
- Doctor imaging assets panel renders a safe state (rows, safe empty, or safe configured error).
- Preview does not expose signed URLs in visible text.
- Logout works and returns to `/login`.
- No forbidden doctor-context tokens are visible:
  - `storageObjectPath`
  - `storage_object_path`
  - `exif`
  - raw storage path

## 5. No-go criteria

- Visible secrets / tokens / signed URLs / storage paths in UI.
- Login unavailable for all users.
- Protected content visible to anonymous users.
- Missing or wrong frontend Supabase env vars (`VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`).
- Failing `npm run preflight:auth-assets`.
- New `deno.lock` file present.
- Unexplained `package-lock.json` churn.

## 6. Decision owner placeholders

Keep real names outside repo. Use placeholders only:

- Release owner: recorded outside repo.
- QA/smoke owner: recorded outside repo.
- Rollback owner: recorded outside repo.
- Security/privacy reviewer: recorded outside repo.

## 7. Post-release follow-up

Record after release:

- Release timestamp.
- Commit SHA.
- Workflow links (`frontend-auth-assets`, `auth-assets-smoke-skip`).
- Smoke result (pass / fail / deferred).
- Incident / no-incident status.
- Follow-up stage created if code or doc changes are needed.
- Confirmation that no secrets were recorded in notes, screenshots, or links.
