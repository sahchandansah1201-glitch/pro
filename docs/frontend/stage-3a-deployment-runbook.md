# Stage 3A — Deployment runbook for auth/assets readiness

## 1. Purpose

- Operational checklist for promoting the completed auth/assets readiness slice.
- Does not change runtime behavior.
- Assumes the Stage 2F release-ready freeze is satisfied.

## 2. Pre-deploy gates

Run and confirm each gate before promoting a build:

- `npm run preflight:auth-assets` ends with `[preflight-auth-assets] OK`.
- `node scripts/check-no-deno-locks.mjs` is clean.
- `git status --short` is clean (or contains only intended release notes).
- `frontend-auth-assets` workflow is green on the target ref.
- `auth-assets-smoke-skip` workflow is green or cleanly skipped as expected.
- `package-lock.json` is preserved (not deleted, not regenerated by hand).
- No `deno.lock` files exist anywhere in the repository.
- No `test-results/` or `playwright-report/` artifacts are committed.

## 3. Required frontend env vars

Configure these in the deploy environment (build-time):

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_PUBLISHABLE_KEY`

Rules:

- `SUPABASE_SERVICE_ROLE_KEY` MUST NOT be present in any frontend env, build env, or bundled code. It is server-only.
- If the `VITE_*` vars are missing, the app remains in demo/unconfigured mode and auth/assets flows will not be live.

## 4. Optional local real-auth smoke

Real-auth smoke is local and opt-in only. Do not run it in CI.

Command:

- `npm run smoke:auth-assets`

Required env:

- `E2E_DOCTOR_EMAIL`
- `E2E_DOCTOR_PASSWORD`
- `E2E_VISIT_ROUTE`

Optional env:

- `E2E_EXPECT_ASSET_ROW=1`
- `E2E_TRY_PREVIEW=1`
- `PW_CHROMIUM_PATH`

Rules:

- The smoke is read-only. It must not upload, modify, or delete assets.
- Do not paste credentials, access tokens, signed URLs, or storage paths into logs, docs, issues, or PR descriptions.

## 5. Deploy verification

After the deploy is live, manually verify on the deployed environment:

- `/login` real auth works with a doctor user.
- A protected doctor route redirects anonymous users to `/login`.
- Login return-to works for an allowed doctor route.
- Doctor visit imaging tab is not in demo mode when auth/env are configured.
- Assets panel can list rows or shows a safe empty/error state.
- Preview opens in-app without exposing a signed URL as visible text.
- Logout returns to `/login`.

## 6. Rollback notes

- Rollback rehearsal details live in docs/frontend/stage-3b-rollback-drill.md.
- Post-deploy smoke details live in docs/frontend/stage-3c-production-smoke.md.
- If auth/assets behavior breaks after deploy, revert the deploy artifact / hosting release.
- Do not "roll back" by deleting `package-lock.json`.
- Do not commit `deno.lock` files during a rollback.
- If the failure is environmental (wrong/missing `VITE_*` vars), prefer fixing environment configuration over a code rollback.
- If Edge Functions or backend are unavailable, the frontend should show safe configured-error states, not raw storage paths or internal identifiers.

## 7. Known non-blocking warnings

These are expected and do not block release:

- React Router v7 future flag warnings.
- Browserslist `caniuse-lite` age warning.
- Vite chunk size warning.

## 8. Release owner checklist

- Preflight green.
- CI workflows green on target ref.
- Required `VITE_*` env vars confirmed in deploy environment.
- Optional real-auth smoke decision recorded (run / skip).
- Rollback owner known.
- No secrets in release notes or logs.
- Final `git status --short` reviewed.
